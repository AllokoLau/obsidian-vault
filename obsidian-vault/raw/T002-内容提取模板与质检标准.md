## 1. 内容提取模板

### 1.1 通用提取字段（所有播客剧集共用）

以下字段从每集播客的 shownotes + 字幕中提取：

| 序号 | 字段名 | 提取方式 | 是否必填 | 来源标注格式 |
|------|--------|----------|----------|-------------|
| 1 | `title` | 播客 API 返回的 `title` 字段 | ✅ 必填 | `[来源:节目标题/原始字段]` |
| 2 | `date` | 播客 API 的 `pubDate` 字段，统一为 ISO 8601 | ✅ 必填 | `[来源:节目发布日期]` |
| 3 | `source_name` | 固定值（晚点聊/厚雪长波/十字路口/张小珺） | ✅ 必填 | `[外部-小宇宙/{播客名}]` |
| 4 | `source_url` | 播客 API 的 `link` 字段 | ✅ 必填 | `[来源:{URL}]` |
| 5 | `shownotes` | 播客 API 的 `description`/`content` 字段 | ✅ 必填 | `[来源:节目Shownotes]` |
| 6 | `transcript` | 小宇宙字幕接口 JSON → 提取纯文本 | ✅ 必填 | `[来源:小宇宙字幕接口]` |
| 7 | `duration` | 播客 API 的 `duration` 字段（秒） | 可选 | `[来源:节目元数据]` |

### 1.2 Founder 类型附加字段

当判定为 founder 类型时，额外提取以下字段 [来源:T001/创始人访谈判定规则] [置信度:0.88]：

| 序号 | 字段名 | 提取方式 | 模板对应字段 |
|------|--------|----------|-------------|
| F1 | `interviewee_name` | 从标题正则提取（11种访谈标题模式）+ shownotes 首段 NER | `姓名` |
| F2 | `company` | 从标题/shownotes 关键词匹配 + 已知公司库 | `创始公司` |
| F3 | `industry` | 基于公司+内容的行业分类（AI大模型/芯片/机器人/消费电子/自动驾驶/投资/...） | `行业` |
| F4 | `position` | 从 shownotes 提取（CEO/CTO/创始人/联合创始人/教授/研究员/...） | `职位` |
| F5 | `career` | 从字幕提取职业经历段落（AI 摘要到 ≤300 字） | `职业经历` |
| F6 | `viewpoints` | 从字幕提取核心观点（每观点 ≤150 字，3-5 条） | `公开观点/言论` |

**姓名提取规则（11 种模式）** [来源:T001/各源标题模式分析]：

```
模式 1: "与{姓名}聊" → 提取 {姓名}
模式 2: "对{姓名}的访谈" → 提取 {姓名}
模式 3: "对话{姓名}" → 提取 {姓名}
模式 4: "{姓名}口述" → 提取 {姓名}
模式 5: "和{姓名}聊" → 提取 {姓名}
模式 6: "{公司}创始人{姓名}" → 提取 {姓名}，同时提取 {公司}
模式 7: "{姓名}的{话题}" → 提取 {姓名}
模式 8: "{公司}{姓名}聊" → 提取 {姓名}，同时提取 {公司}
模式 9: "访{姓名}" → 提取 {姓名}
模式 10: "与{公司}{姓名}聊" → 提取 {姓名}，同时提取 {公司}
模式 11: 标题含已知姓名库匹配 → 直接提取
```

**⚠️ 提取失败时的处理**：如以上 11 种模式均未匹配到姓名，不得留空或填"公司:"。应将该剧集降级为 `knowledge` 类型 [置信度:0.60]。

### 1.3 Knowledge 类型附加字段

当判定为 knowledge 类型时，额外提取以下字段 [来源:T001/各源内容结构]：

| 序号 | 字段名 | 提取方式 | 模板对应字段 |
|------|--------|----------|-------------|
| K1 | `topic` | 从标题提取话题名称 | `主题` |
| K2 | `category` | 基于内容匹配：投资分析 / AI学习 / 科技创业 / 宏观经济 | `分类` |
| K3 | `scenario` | AI 从内容推断适用场景（1-2 句话） | `适用场景` |
| K4 | `core_conclusion` | AI 摘要 3-5 句核心结论 | `核心结论` |
| K5 | `key_points` | 提取 3-5 个要点（每个 ≤200 字） | `要点` |
| K6 | `practice_advice` | 如有实践建议则提取，否则留空 | `实践建议` |

### 1.4 类型判定决策树

```
输入：播客剧集标题 + shownotes
│
├─ 标题匹配 11 种 founder 模式？
│  ├─ YES → 提取到姓名？→ YES → type: founder [置信度:≥0.80]
│  │                    → NO  → type: knowledge [置信度:0.60]
│  └─ NO  → shownotes 含"创始人/CEO/CTO/联合创始人"？
│           ├─ YES → 尝试 NER 提取姓名 → 成功 → type: founder [置信度:0.70]
│           │                           → 失败 → type: knowledge [置信度:0.60]
│           └─ NO  → 标题/内容是否纯话题讨论（无明确个人）？
│                    ├─ YES → type: knowledge [置信度:0.80]
│                    └─ 不确定 → type: knowledge + 标注盲区 [置信度:0.50]
```

## 2. 语言优化规范

以下规范应用于所有入库内容 [基于 Writer 4 重质检中语言优化环节]：

### 2.1 术语统一

| 不规范用法 | 统一用法 | 说明 |
|-----------|---------|------|
| AI/人工智能 | **AI** | 首次出现可写"AI（人工智能）"，之后统一用 AI |
| 大模型/LLM/语言模型 | **大模型** | 上下文明确时可用 LLM |
| 创始人/CEO/老板 | 按实际职位填写 | 不得混用 |
| AGI/通用人工智能 | **AGI** | 首次可注全称 |
| 具身智能/具身/Embodied AI | **具身智能** | |
| Agent/智能体/AI代理 | **Agent** | 中文语境可用"AI Agent" |

### 2.2 文本缩减规则

- 原文 10000+ 字 → 缩减到 800-1500 字（保留信息密度，删除口语化填充词）
- 每个"公开观点/言论"条目 ≤150 字
- 职业经历 ≤300 字
- 核心结论 3-5 句，共 ≤200 字

### 2.3 格式规范

- 中文与英文/数字之间加半角空格：`AI 大模型`、`500 亿参数`、`GPT-5`
- 中文与行内代码之间加空格：`用 `publish_image()` 发布`
- 引用的书名/文章名用《》
- 播客/节目名保持原名不加书名号

## 3. 4 重质检清单

以下为每一条入库 Wiki 内容必须通过的质检标准。

### 3.1 ① source_citation（来源引用）

| 检查项 | 方法 | 通过标准 |
|--------|------|----------|
| 每条事实性陈述是否有来源标注 | 全文搜索 `[来源:` | 100% 覆盖 |
| 来源标签格式是否正确 | 检查 `[外部-小宇宙/{播客名}]` 格式 | 不符合则修正 |
| 来源 URL 是否有效 | 抽样点击验证 | 90%+ 可访问 |
| 是否标注了置信度 | 搜索 `[置信度:` | High/Medium/Low 或 0.XX 格式 |

### 3.2 ② factual_consistency（事实一致）

| 检查项 | 方法 | 通过标准 |
|--------|------|----------|
| 姓名/公司/职位与原始材料一致 | 逐条对照 Patricia 研究材料 | 完全一致 |
| 数字/日期/金额无转录错误 | 与节目 shownotes/字幕原文对比 | 无偏差 |
| 无 Writer 自行添加的新事实 | 全文对比 Patricia 材料 | 无新增事实 |

### 3.3 ③ research_backcheck（回查）

| 检查项 | 方法 | 通过标准 |
|--------|------|----------|
| 核心结论是否能被其他独立来源验证 | 对核心结论反向检索 Wiki 数据层/网络 | 至少 1 个独立来源支持 |
| 是否有 Patricia 未覆盖的新信息 | 检查是否有时效性更强的信息 | 如有则标注 `[需补充研究]` |

### 3.4 ④ metadata_complete（元数据完整）

| 检查项 | 方法 | 通过标准 |
|--------|------|----------|
| YAML frontmatter 含全部必填字段 | 按模板逐字段检查 | 必填字段无空值 |
| `type` / `入库日期` / `置信度` / `tags` 已填 | 人工确认 | ✓ |
| Founder: `姓名` `公司` `行业` `职位` 已填 | 人工确认 | 无"待补充" |
| Knowledge: `主题` `分类` `来源` 已填 | 人工确认 | 无空值 |
| 文件命名符合规范 | Founder: `姓名-公司.md` / Knowledge: `日期_主题.md` | 符合 |
