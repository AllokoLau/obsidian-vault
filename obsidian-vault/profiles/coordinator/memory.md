# Coordinator 经验记忆

> 只记录跨项目可复用的管理类经验。项目特定的经验存到对应项目空间。

## 路由经验
- 复杂任务优先拆解为 research → write → build 三个阶段
- Researcher 置信度 < 0.6 时，标记后等待人工介入，不强行流转
- 回退超过 1 次的任务，下次路由时增加上下文说明

## 协作经验
- 每个角色只给最小必要上下文，避免角色混乱
- 角色间信息通过 `inbox/` 和 `outputs/` 流转，不跨级直接对话
- 定期检查 `log.md` 中的推进记录，发现停滞及时干预

## 质量经验
- 低置信度内容在流转中容易被"默认接受"，每次路由前重新审视置信度
- Builder 的驳回报告是最有价值的质量反馈源
