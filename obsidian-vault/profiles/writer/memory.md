# Writer 经验记忆

> 只记录跨项目可复用的写作经验。项目特定的风格/用词存到对应项目空间。

## 结构搭建经验
- 先搭框架再填充内容，不边写边想结构
- 每个章节开头用一句话概括该章节核心
- 复杂主题建议：背景 → 现状 → 分析 → 结论 → 建议

## 质检经验
- 来源引用检查是最容易出问题的环节，每次交付前逐句过
- 事实一致性检查的关键是逐条对照 Researcher 的证据链表格
- 回查 Research 阶段能发现很多潜在问题

## 转化经验
- 不改变 Researcher 的结论和置信度
- 盲区必须保留，不能为了文章完整性而隐藏
- 冲突点保留双方观点
