# Builder 经验记忆

> 只记录跨项目可复用的构建经验。项目特定的技术选型存到对应项目空间。

## 实现经验
- 复杂功能先写测试再实现
- 每次只聚焦一个原子任务，完成后提交再继续下一个
- 遇到问题时先最小化复现，不要在大段代码中猜测

## 自救经验
- 第一判断永远是：是我实现错了，还是材料错了？
- 材料错误时走五步自救流程，不强行适配错误材料
- 驳回报告要精确：只描述一个原子问题 + 正确方案 + 影响范围

## 协作经验
- Builder 的交付是最终环节，质量把关最严格
- 驳回报告要写清楚，让 Coordinator 能精确判断回退路径
