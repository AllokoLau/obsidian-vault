## 正文

### DAA 指标的提出

2026 年 6 月的 Create 大会上，百度创始人李彦宏正式提出 DAA（Daily Active Agents，日活智能体数）概念，将其定义为衡量 AI 时代价值的新度量衡。这一指标的核心主张是：不要再数有多少人在用 AI，而要数有多少智能体在替人干活、并交付了结果。

DAA 的提出并非孤立行为，而是李彦宏自 2024 年以来智能体战略思想的逻辑延伸。2024 年 7 月 WAIC 大会上，他就提出要"避免掉入超级应用陷阱"，认为出现 10 亿 DAU 的 App 才是成功是移动时代的思维逻辑。2024 年底百度世界大会上，他进一步强调"智能体是 AI 应用的最主流形态"，并发布了秒哒——一个无需写代码即可生成应用的平台。2025 年 11 月，百度推出"伐谋"智能体，借鉴进化算法实现自我迭代，"效果涌现"和"AI 内化"成为新关键词。

### 从 Token 经济学到 DAA

黄仁勋提出的 Token 经济学将 Token 消耗量视为 AI 时代的 GDP。李彦宏对此提出直接质疑："Token 只是代表成本，并不代表收益；它衡量的是投入，而不是产出。" 在他看来，只看 Token 消耗就像只看一个国家的电力消耗量来评估经济水平——有相关性，但远远不够。当 agent 能自主执行任务、在执行中学习、在学习中变强，衡量尺度就必须从投入端转向产出端。

DAA 隐含的逻辑框架是：智能体 "自我进化" 的程度应成为行业核心指标，而非模型参数或 Token 消耗量。

### 智能体产品矩阵

**DuMate**（百度 3 月上线的通用智能体）被定位为"第二代入口"——不同于 ChatGPT 为代表的聊天机器人（解决信息获取），DuMate 解决的是任务完成问题。它本身不内置所有功能，而是作为 agent 调度器，调用秒哒、伐谋、搜索、文件管理等独立智能体协同工作。在典型运营场景中，DuMate 可同时处理客服、数据分析和营销三条线，百度内部称之为"人机混编"。

**秒哒** 面向零代码用户，上线以来已服务超 1000 万用户，81% 为非技术人员。3.0 版本支持直接生成可部署 App 及多人协作开发。Create 现场展示了 8 岁温州小学生用秒哒制作"哒哒打伞"拼伞 App 的案例。

**伐谋 2.0** 面向专业决策者，通过持续对话式推演帮企业找到复杂问题的最优解。青岛港引入后，已连续刷新世界纪录的自动化码头调度系统 A-TOS 实现了 10.21% 的进一步突破。

**一镜**（原慧播星）通过多 agent 协同（剧本 agent + 视频 agent + 剪辑 agent）实现全场景数字人内容生产。

### 全栈整合共识

DAA 背后是百度"芯云模体"全栈战略。这与全球 AI 巨头的动作方向高度一致：Google Cloud Next 2026 发布第八代 TPU 双芯片；Anthropic 与 SpaceX 签约算力协议；OpenAI 的 Codex 从代码工具转向 agent 平台；微软上线 Agent 365 企业级平台。四家公司的动作指向同一共识：做好智能体需要从芯片到云再到平台调度层的全套端到端优化。

百度的昆仑芯 P800 已完成万片级交付，256 卡超节点计划 6 月问世。

### 挑战与前景

DAA 能否从百度的一家之言变成行业语言，取决于三个前提：agent 产品本身的成熟度（当前多数 agent 仍处于 Demo 到稳定产品的过渡期）；DAA 定义的可量化性（如何标准化衡量智能体交付的成果）；以及行业对"产出端指标"的共识度。

李彦宏在大会上做出一个深远的判断：当每个人都有成倍于之前的效率后，金字塔型管理结构将不再必要，管理的主要功能将从监督、指挥、激励变为对齐——确认他人在做对的事。DAA 不只是产品指标，而是关于劳动与组织全新形态的一种认知。
