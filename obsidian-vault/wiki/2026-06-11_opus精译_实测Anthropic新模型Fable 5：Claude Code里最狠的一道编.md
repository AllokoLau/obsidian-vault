### 关键要点

1. **Fable 5 的自主编码能力**：在 Claude Code 中无需人工干预，Agent 自主完成了从方案设计到编码实现的全流程。作者使用了 planning mode（规划模式）先生成方案，再用 goal 命令拆解为功能文件并逐个实现，最后挂载后台审查 Agent 把关。

2. **推理强度选择策略**：Anthropic 官方推荐大部分任务使用 "high" 级别，但作者选用了更高的 "ultra-high"（极高级别）。他认为最高档（extreme）可能导致边际效益递减，模型可能胡言乱语并自我误导。

3. **RAMP 框架的应用**：作者使用自己的 RAMP 开发框架来组织编码流程——将 plan 转化为 detailed implementation plan，再拆分为独立的功能文件（functional files），每个文件都极其详细，便于 goal 命令逐文件实现。

4. **性能与成本权衡**：后台审查选择 Opus 4.8 而非 Fable 5 以节省 token 消耗。Fable 5 定价是 Opus 4.8 的两倍（$10/$50 vs 约 $5/$25 per M tokens），2 小时编码消耗了约 50 万 token。作者认为虽然昂贵，但对于复杂推理任务可能值得。

5. **成品质量评估**："镜界熔炉"实现了真实的反射效果（镜面反射球体、激光）、光线追踪（激光碰到物体停止、在镜面反弹）、用户交互（点击镜面面板旋转、Q/E 键旋转视角），所有核心功能均正常工作。

6. **与 GPT 5.5 的对比**：GPT 5.5 运行时间更短（1h20min vs 2h），生成的成品也包含反射效果和光线追踪，但反射精度和整体效果不及 Fable 5。作者评价："效果也不错，但肯定不如这个"。
