# 为什么「高价值任务」成了所有 AI Labs 的T0 级战略？

## 核心摘要

1. 硅谷正经历"Move 37 时刻"：几乎所有 AI 工程师亲手写代码的比例从 70-80% 降到 5% 以下，很多人连代码编辑器都不再打开。变化从 Claude Opus 4.5 发布和 Claude Code Harness 提升两件事叠加而生。
2. "高价值任务"已成为所有 AI Labs 的 T0 级战略。Anthropic ARR 从 2025 年底约 90 亿美元飙升至 2026 年 4 月约 300 亿美元，三个月翻 3 倍。Coding 从"重要场景之一"变成几乎所有 Labs 的最高优先级。
3. 新时代的价值公式从 "DAU x ARPU" 切换为 "Token 消耗 x 任务单价"。Claude Code 用 ChatGPT 约 5% 的用户量跑出可比收入，在互联网时代不成立。

5. 二八结构在当下成立依赖于顶尖模型在 Planner 位置上拉得开差距。一旦平价模型跨过够用阈值，顶尖模型将从"80% spending 区"滑向"80% token 区"。

## 正文

### 硅谷的"Move 37 时刻"

2025 年的技术变化偏线性，但 2026 年 3 月的硅谷体感完全不同。几乎所有 AI 工程师和 AI 研究员在过去 1-2 个月内从写代码变成只做 review，有人直接承认 Claude Code 比自己强，亲手写代码比例从七八十降到 5% 以下。

这种冲击一定会延到 B 端：当每个员工都能在公司内部搓一套 CRM 时，企业软件的毛利面临重估。白领被替代（seat-based 需求收缩）只是第一层，更严重的是企业软件毛利本身被重估。

两件同时发生的突破让模型公司战略收敛：
1. Claude Opus 4.5 发布——one-shot prompt 基本就能完成任务。
2. Claude Code 的 Harness 本身在提升——为 agent 搭建的工作环境越来越适合它自己发挥。

### 高价值任务：T0 级战略

2025 年观察 Labs 策略时关键词是"分化"（Anthropic 像微软、OpenAI 像 Google），但今天所有 Labs 策略又回到高度收敛的方向——高价值任务。

"高价值任务"的关键在于端到端可以完成任务——不是"我能获得讯息"，而是"我真的能 take action"。当模型和 agent 真正开始 take actions 时，全球白领的工资池就会被重新分配。法律、金融、Coding 等高薪工种共同点是高智力从业门槛、单位智能消耗高、单位产出价值高。

### ToC/ToB 是互联网时代的旧地图

用户量不再是价值的充分条件。Claude Code 用户规模大约只有 ChatGPT 的 1/20，但两者 ARR 基本持平，甚至有反超趋势（截至 2026 年 4 月，Anthropic ARR 约 300 亿美元，超过 OpenAI 约 250 亿美元）。

新时代价值公式从 "DAU x ARPU" 切换为 "Token 消耗 x 任务单价"。B 端形态也在变——当白领被 AI 替代后，B 端人头规模缩小但单位价值和 adoption 速度都在变大。SaaS 按 seat 定价正在被按 outcome 定价替代。

Agent 时代的真正维度不是 ToC 还是 ToB，而是它承载的是不是高价值任务、分到的是 80% 的 token 还是 80% 的 spending。

### 25% 的卡创造 80% 的收入

GTC 2026 上 Jensen Huang 定义高价值任务：由 agent 独立完成、长周期、吐出大量 token 最终交付结果的任务。NVDA 把模型使用场景分成四档，每档平均分配 25% 的卡。最高价值的 25% 使用 GPU + Grok LPU 新方案，虽然只用了 1/4 的卡，却可能为客户创造 80% 的收入。

这张图的真正含义不在硬件本身，而在于对高价值任务做了一次物理层面的背书：只要高价值任务必须跑在最快的硬件加最好的模型上，顶尖模型就有结构性的定价能力。

### 硅谷开始投芯片

过去两个月湾区 VC 出现"反常"趋势——硅谷过去二三十年的钱大部分投在软件上，最近却纷纷开始投芯片。Cerebras 把芯片做得很大，NVDA 收购 Grok 并用 SRAM 设计新芯片。核心原因是 Agent 时代（尤其 multi-agents）需要 AI-native hardware，对 context 处理能力、单次推理速度、数据 throughput 同时提出更高要求。

### 80% 的 token 与 80% 的 spending 是两件事

Jensen 的思路是把高价值档位往上拉，另一条线是 OpenClaw——让便宜模型通过更好的 Harness 够到高价值任务上限。OpenClaw 赢得市场在于创新 ideas，但真正把 Harness 做到生产级的，大概率还是围绕 Claude、OpenAI 等离模型更近的公司。

Multi-agent 的本质是在单位时间内做足够多并行计算，不是算法问题，更多是管理学问题。Multi-agents 里真正值钱的是 planning agent，定角色、开 agent、派任务的核心角色。
