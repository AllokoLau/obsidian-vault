# 来源标签规范

> **层级**: 宪法层  
> **变更频率**: 低（新来源类型需登记）  
> **作用**: 确保所有事实可追溯，统一来源引用格式

---

## 1. 标签格式

所有来源标签使用方括号 `[]` 包裹，格式为：
```
[来源类型-子类型/标识]
```

---

## 2. 标签类型体系

### 2.1 Wiki 内部来源

| 标签格式 | 含义 | 示例 |
|----------|------|------|
| `[Wiki-宪法层/文件名]` | 宪法层规则文件 | `[Wiki-宪法层/角色定义.md]` |
| `[Wiki-运行时层/项目名/文件名]` | 当前或历史项目运行时材料 | `[Wiki-运行时层/项目A/任务清单.md]` |
| `[Wiki-数据层/方法论/文件名]` | 数据层方法论模板 | `[Wiki-数据层/方法论/研究结论模板.md]` |
| `[Wiki-数据层/原始素材/文件名]` | 数据层原始素材 | `[Wiki-数据层/原始素材/文献索引.md]` |

### 2.2 外部来源

| 标签格式 | 含义 | 示例 |
|----------|------|------|
| `[外部-Arxiv/论文ID]` | Arxiv 论文 | `[外部-Arxiv/2401.12345]` |
| `[外部-DOI/DOI号]` | 学术期刊（DOI） | `[外部-DOI/10.1145/123456]` |
| `[外部-GitHub/仓库/路径]` | GitHub 仓库 | `[外部-GitHub/user/repo/blob/main/...]` |
| `[外部-官方文档/产品名/章节]` | 官方文档 | `[外部-官方文档/React/ Hooks]` |
| `[外部-权威媒体/文章URL]` | 权威媒体报道 | `[外部-权威媒体/nytimes.com/...]` |
| `[外部-专业博客/作者/文章]` | 专业博客 | `[外部-专业博客/作者名/文章标题]` |
| `[外部-数据集/名称]` | 公开数据集 | `[外部-数据集/IMDB Reviews]` |
| `[外部-工具/名称/版本]` | 工具/软件 | `[外部-工具/Node.js/20.0.0]` |

### 2.3 历史项目来源

| 标签格式 | 含义 | 示例 |
|----------|------|------|
| `[历史-项目ID/文件名]` | 已归档历史项目材料 | `[历史-PRJ-2024-001/复盘记录.md]` |

### 2.4 方法论来源

| 标签格式 | 含义 | 示例 |
|----------|------|------|
| `[方法论-模板名]` | 数据层方法论模板 | `[方法论-研究结论模板]` |
| `[方法论-框架名]` | 可复用框架 | `[方法论-五步自救流程]` |

## 3. 使用规则

### 3.1 必须标注的场景
- 所有事实性陈述（数据、定义、结论、引用）
- 所有方法论引用（"根据...方法"）
- 所有历史案例引用

### 3.2 可省略标注的场景
- 数学公式的数学推导（但公式来源需标注）
- 角色自身的逻辑推理（如"因此..."、"由此可得..."）
- 常识性表述（但"常识"的界定以术语词典为准）

### 3.3 冲突标注
当多个来源对同一命题给出不同结论时：
```markdown
命题X [来源A:XXX] [置信度:0.85] vs [来源B:YYY] [置信度:0.72]
```

---

## 4. 标签登记

新增来源类型需在本文档中登记，并说明：
- 标签格式
- 适用场景
- 权威性权重（参考 置信度分级标准）

---

## 5. 示例段落

```markdown
根据现有研究，Transformer 架构在自然语言处理任务中表现优异
[来源:外部-Arxiv/1706.03762] [置信度:0.95]。

然而，在资源受限场景下，其计算效率存在挑战
[来源:外部-Arxiv/2009.00031] [置信度:0.82]，
这与我们历史项目中的观察一致 [来源:历史-PRJ-2024-001/复盘记录.md]。
```
