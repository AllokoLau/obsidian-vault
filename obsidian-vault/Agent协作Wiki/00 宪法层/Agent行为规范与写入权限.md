# Agent 行为规范与写入权限

> **层级**: 运行时层 / 全局规则  
> **约束对象**: 所有接入 Obsidian 的 Agent（Coordinator、Researcher、Writer、Builder）  
> **执行方式**: Agent 每次写入前必须自检，本地桥接器二次校验

---

```yaml
---
role: Coordinator
type: 全局规则
doc_name: Agent行为规范与写入权限
version: 1.0
enforced_by: 本地桥接器 + Agent自检
violation_action: 拒绝写入并记录审计日志
---
```

## 1. 核心原则

1. **路径隔离**：每个角色只能写入指定目录，越界视为污染
2. **只读保护**：宪法层（00）和数据层（20）对所有 Agent 只读
3. **版本递增**：每次修改必须更新 `version` 字段
4. **时间戳强制**：每次写入必须附带 ISO 8601 时间戳
5. **覆盖预警**：覆盖未归档文件前必须确认，禁止静默覆盖

## 2. 角色权限矩阵

### 2.1 路径权限总表

| 角色 | 读取范围 | 写入范围 | 绝对禁止 |
|------|----------|----------|----------|
| **Coordinator** | 全部三层 | `10 运行时层/项目空间/{项目}/`（项目目标书、任务清单、上下文摘要、复盘记录）<br>`10 运行时层/全局规则/路由决策日志/`<br>`10 运行时层/全局规则/调度指令存档/` | `00 宪法层/`<br>`20 数据层/`<br>`研究材料/`<br>`写作成品/`<br>`构建产物/`<br>`驳回报告/` |
| **Researcher** | 全部三层 | `10 运行时层/项目空间/{项目}/研究材料/` | `00 宪法层/`<br>`20 数据层/`<br>`写作成品/`<br>`构建产物/`<br>`驳回报告/`<br>`任务清单`（除状态字段外）<br>`项目目标书` |
| **Writer** | 全部三层 | `10 运行时层/项目空间/{项目}/写作成品/`<br>`10 运行时层/全局规则/质检记录/` | `00 宪法层/`<br>`20 数据层/`<br>`研究材料/`<br>`构建产物/`<br>`驳回报告/` |
| **Builder** | 全部三层 | `10 运行时层/项目空间/{项目}/构建产物/`<br>`10 运行时层/项目空间/{项目}/驳回报告/`<br>`10 运行时层/全局规则/质检记录/` | `00 宪法层/`<br>`20 数据层/`<br>`研究材料/`<br>`写作成品/`<br>`项目目标书`<br>`任务清单` |

### 2.2 权限判定逻辑（伪代码）

```
function can_write(role, target_path):
    if target_path contains "00 宪法层/":
        return FALSE
    if target_path contains "20 数据层/":
        return FALSE

    if role == "Coordinator":
        if target_path contains "10 运行时层/项目空间/":
            filename = get_filename(target_path)
            if filename in ["项目目标书.md", "任务清单.md", "上下文摘要.md", "复盘记录.md"]:
                return TRUE
        if target_path contains "10 运行时层/全局规则/路由决策日志/":
            return TRUE
        if target_path contains "10 运行时层/全局规则/调度指令存档/":
            return TRUE
        return FALSE

    if role == "Researcher":
        if target_path contains "10 运行时层/项目空间/" AND target_path contains "/研究材料/":
            return TRUE
        return FALSE

    if role == "Writer":
        if target_path contains "10 运行时层/项目空间/" AND target_path contains "/写作成品/":
            return TRUE
        if target_path contains "10 运行时层/全局规则/质检记录/":
            return TRUE
        return FALSE

    if role == "Builder":
        if target_path contains "10 运行时层/项目空间/" AND target_path contains "/构建产物/":
            return TRUE
        if target_path contains "10 运行时层/项目空间/" AND target_path contains "/驳回报告/":
            return TRUE
        if target_path contains "10 运行时层/全局规则/质检记录/":
            return TRUE
        return FALSE

    return FALSE
```

## 3. 禁止行为清单

### 3.1 一级禁止（立即拒绝，记录审计）

| 编号 | 禁止行为 | 触发条件 | 后果 |
|------|----------|----------|------|
| P-001 | 写入宪法层 | 目标路径包含 `00 宪法层/` | 拒绝写入，返回 403，记录审计日志 |
| P-002 | 写入数据层 | 目标路径包含 `20 数据层/` | 拒绝写入，返回 403，记录审计日志 |
| P-003 | 越界写入其他角色专属区 | Researcher 写 `写作成品/`，Writer 写 `构建产物/` 等 | 拒绝写入，返回 403，记录审计日志 |
| P-004 | 删除未归档文件 | 目标文件存在且未标记 `status: 已归档` | 拒绝删除，返回 403，通知 Coordinator |
| P-005 | 修改他人正在执行中的任务 | 任务清单中该任务状态为 `执行中` 且负责人不是当前角色 | 拒绝修改，返回 403 |

### 3.2 二级禁止（警告，允许覆盖但需标注）

| 编号 | 禁止行为 | 触发条件 | 后果 |
|------|----------|----------|------|
| P-101 | 无 YAML 头部写入 | 文件内容不以 `---` 开头 | 警告，自动注入默认 YAML |
| P-102 | YAML 缺少强制字段 | 缺少 `role`、`timestamp`、`version` 之一 | 警告，自动补全缺失字段 |
| P-103 | 时间戳格式错误 | `timestamp` 不是 ISO 8601 格式 | 警告，自动修正为当前时间 |
| P-104 | 版本号未递增 | 新文件 version 不大于旧文件 | 警告，自动修正 version = 旧version + 1 |
| P-105 | 术语不一致 | 内容中出现术语词典未定义的术语 | 警告，不阻断，但记录术语偏差日志 |

### 3.3 三级禁止（建议，不阻断）

| 编号 | 禁止行为 | 触发条件 | 后果 |
|------|----------|----------|------|
| P-201 | 来源标签格式错误 | `[来源:XXX]` 格式不符合规范 | 建议修正，记录格式问题 |
| P-202 | 置信度标注缺失 | 结论性陈述缺少 `[置信度:0.XX]` | 建议补充，记录缺失位置 |
| P-203 | 文件命名不规范 | 文件名不符合 `{任务ID}-{描述}.md` 格式 | 建议重命名，记录不规范文件名 |

## 4. 写入前自检清单（Agent 必须执行）

Agent 每次调用写入工具前，必须按顺序执行以下检查：

```markdown
### 写入前自检清单

- [ ] **路径检查**
  - [ ] 目标路径是否在我的可写范围内？（对照权限矩阵）
  - [ ] 目标路径是否包含 `00 宪法层/` 或 `20 数据层/`？（是则停止）
  - [ ] 目标路径是否属于其他角色的专属区？（是则停止）

- [ ] **文件检查**
  - [ ] 目标文件是否已存在？
  - [ ] 如果已存在，是否已归档（`status: 已归档`）？
  - [ ] 如果未归档，我是否有权限覆盖？（Coordinator 可覆盖全部，其他角色只能覆盖自己的）

- [ ] **YAML 检查**
  - [ ] 是否包含 `role` 字段？（值为当前角色名）
  - [ ] 是否包含 `timestamp` 字段？（ISO 8601 格式）
  - [ ] 是否包含 `version` 字段？（大于旧版本号）
  - [ ] `fileClass` 是否与文件类型匹配？

- [ ] **内容检查**
  - [ ] 术语是否与 `00 宪法层/术语词典.md` 一致？
  - [ ] 事实性陈述是否有 `[来源:XXX]` 标注？
  - [ ] 结论是否有 `[置信度:0.XX]` 标注？

- [ ] **命名检查**
  - [ ] 文件名是否符合 `{任务ID}-{描述}.md` 或 `{类型}{序号}-{任务ID}-{日期}.md`？
```

**全部勾选通过 → 允许写入**  
**任何一级禁止项未通过 → 拒绝写入，返回错误信息**  
**二级禁止项未通过 → 警告后自动修正，允许写入**  
**三级禁止项未通过 → 记录建议，允许写入**

## 5. YAML 头部强制规范

### 5.1 强制字段

每个写入的文件必须在头部包含以下 YAML 字段：

```yaml
---
role: Coordinator              # 写入者角色（必填）
timestamp: 2026-06-23T19:49:00+08:00  # 写入时间，ISO 8601，带时区（必填）
version: 1.0                   # 版本号，每次修改+1（必填）
project_id: PRJ-2026-001       # 所属项目ID（项目相关文件必填）
task_id: T001                  # 关联任务ID（任务相关文件必填）
fileClass: 任务清单             # Metadata Menu 识别用（必填）
---
```

### 5.2 字段校验规则

| 字段 | 类型 | 校验规则 | 自动修正 |
|------|------|----------|----------|
| `role` | 字符串 | 必须在 [Coordinator, Researcher, Writer, Builder] 中 | 否，缺失则拒绝写入 |
| `timestamp` | 字符串 | 必须符合 ISO 8601，如 `2026-06-23T19:49:00+08:00` | 是，格式错误时自动替换为当前时间 |
| `version` | 字符串/数字 | 必须大于该文件旧版本的 version | 是，自动递增 |
| `project_id` | 字符串 | 格式 `PRJ-YYYY-NNN` | 否，缺失时警告 |
| `task_id` | 字符串 | 格式 `T{NNN}` | 否，缺失时警告 |
| `fileClass` | 字符串 | 必须与 `fileClass/` 下的文件名之一匹配 | 否，缺失时警告 |

## 6. 文件命名规范

### 6.1 项目文件夹命名
```
项目{项目名}-{PRJ-YYYY-NNN}/
```
示例：`项目Python框架调研-PRJ-2026-001/`

### 6.2 任务文件命名
```
{任务ID}-{描述}.md
```
示例：`T001-框架现状研究.md`、`T003-调研报告.md`

### 6.3 路由决策日志命名
```
R{序号}-{任务ID}-{YYYYMMDD}.md
```
示例：`R001-T001-20260623.md`

### 6.4 质检记录命名
```
Q{序号}-{成品ID}-{YYYYMMDD}.md
```
示例：`Q001-T003-20260623.md`

### 6.5 驳回报告命名
```
BR{序号}-{任务ID}-{YYYYMMDD}.md
```
示例：`BR001-T004-20260623.md`

### 6.6 复盘记录命名
```
review-{PRJ-YYYY-NNN}-{YYYYMMDD}.md
```
示例：`review-PRJ-2026-001-20260623.md`

## 7. 违规处理流程

```
Agent 发起写入请求
    ↓
本地桥接器接收请求
    ↓
执行权限校验（对照权限矩阵）
    ↓
    ├─ 一级禁止触发 → 拒绝写入（403）→ 记录审计日志 → 通知 Coordinator
    ├─ 二级禁止触发 → 警告 → 自动修正 → 允许写入 → 记录修正日志
    ├─ 三级禁止触发 → 建议 → 允许写入 → 记录建议日志
    └─ 全部通过 → 允许写入（200）→ 记录正常日志
    ↓
写入文件到 Vault
    ↓
Obsidian 自动刷新
    ↓
Git 自动 commit（5分钟间隔）
```

### 7.1 审计日志格式

违规时，在 `10 运行时层/全局规则/审计日志/` 下创建文件：

```yaml
---
log_id: AUD-20260623-001
timestamp: 2026-06-23T19:49:00+08:00
role: Researcher
violation_type: P-003
violation_desc: 试图写入写作成品目录
target_path: 10 运行时层/项目空间/.../写作成品/T001-越界.md
action: 拒绝写入
status: 已处理
---

## 详情
Agent 尝试以 Researcher 身份写入 Writer 专属区，已被本地桥接器拦截。

## 建议
检查 Researcher 的 Prompt 中的路径规范说明，确认是否明确禁止写入非研究材料目录。
```

## 8. Agent Prompt 追加规范

每个角色的系统 Prompt 末尾必须追加以下内容：

```markdown
## Obsidian 写入规范（强制执行）

### 你的可写范围
[根据角色填入对应路径，如 Researcher 只能写 `10 运行时层/项目空间/{项目}/研究材料/`]

### 每次写入前必须执行
1. 确认目标路径是否在你的可写范围内
2. 确认目标路径不包含 `00 宪法层/` 或 `20 数据层/`
3. 确认 YAML 头部包含 `role`、`timestamp`、`version`
4. 确认 version 大于该文件的旧版本号

### 禁止行为（违反将被拒绝写入）
- 禁止写入 `00 宪法层/` 和 `20 数据层/`
- 禁止写入其他角色的专属文件夹
- 禁止覆盖未归档的文件（除非你是 Coordinator）
- 禁止删除任何文件

### 命名规范
- 研究材料：`{任务ID}-{描述}.md`
- 写作成品：`{任务ID}-{描述}.md`
- 构建产物：`{任务ID}-{描述}.py`（或对应扩展名）
- 驳回报告：`BR{序号}-{任务ID}-{日期}.md`

### YAML 头部模板
```yaml
---
role: [你的角色名]
timestamp: [当前时间，ISO 8601]
version: [版本号]
project_id: [项目ID]
task_id: [任务ID]
fileClass: [对应fileClass名]
---
```
```
