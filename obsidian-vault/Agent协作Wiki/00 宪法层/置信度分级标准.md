# 置信度分级标准

> **层级**: 宪法层  
> **变更频率**: 极低  
> **作用**: 统一所有角色的置信度评估基准

---

## 1. 置信度定义

置信度（Confidence Score）是一个 0-1 之间的数值，表示对某一命题、证据或结论的确信程度。

- **0.0**: 完全不可信 / 已知错误
- **0.5**: 无法判断，信息不足
- **1.0**: 完全确信，多源交叉验证，无争议

---

## 2. 分级标准

### A级：高度确信（0.90 - 1.00）

**判定标准**（满足全部）：
- [ ] 至少 2 个独立来源交叉验证
- [ ] 来源权威性高（同行评审、官方文档、权威媒体）
- [ ] 无已知冲突或冲突已解决
- [ ] 时效性符合要求

**路由动作**：直接路由到下一角色，无需干预

---

### B级：较确信（0.70 - 0.89）

**判定标准**（满足其一）：
- [ ] 单一高权威性来源，但无交叉验证
- [ ] 多来源验证，但存在轻微不确定性（如数据口径差异）
- [ ] 来源权威性中等，但逻辑自洽

**路由动作**：路由到下一角色，但标记"建议复核"

---

### C级：存疑（0.50 - 0.69）

**判定标准**（满足其一）：
- [ ] 来源有限，仅 1 个低权威性来源
- [ ] 多来源存在可解释的冲突
- [ ] 信息部分缺失，但核心命题可推断

**路由动作**：回退到 Researcher 补充调研，达标后再路由

---

### D级：不可信（0.00 - 0.49）

**判定标准**（满足其一）：
- [ ] 信息严重不足，无法支撑任何结论
- [ ] 来源可信度低（匿名、未验证、已知偏见）
- [ ] 多来源存在不可调和的冲突
- [ ] 已知与已验证事实矛盾

**路由动作**：驳回当前任务，Coordinator 重新分析需求或要求用户澄清

---

## 3. 置信度计算规则

### 3.1 单条证据置信度
```
证据置信度 = 来源权威性权重 × 时效性权重 × 验证状态权重
```

| 维度 | 权重参考 |
|------|----------|
| 来源权威性 | 同行评审(1.0) > 官方文档(0.95) > 权威媒体(0.9) > 专业博客(0.7) > 社交媒体(0.4) |
| 时效性 | 最新(1.0) > 1年内(0.95) > 3年内(0.85) > 5年内(0.7) > 过时(0.5) |
| 验证状态 | 已验证(1.0) > 部分验证(0.8) > 未验证(0.6) > 存疑(0.4) |

### 3.2 综合结论置信度
```
结论置信度 = min(证据链平均置信度, 最弱证据置信度 × 1.2)
```
> 取平均值和最弱证据的加权值中的较小者，确保短板效应

### 3.3 降级规则
- 存在未标注的盲区 → 最高置信度不得超过 0.79
- 存在冲突点但未处理 → 最高置信度不得超过 0.69
- 来源未标注 → 最高置信度不得超过 0.49

## 4. 使用场景

| 场景 | 评估主体 | 动作 |
|------|----------|------|
| 任务拆分预估 | Coordinator | 预估每个任务的 estimated_confidence |
| 研究结论 | Researcher | 对每条证据和整体结论打分 |
| 成品质检 | Writer | 4重质检中的整体置信度评估 |
| 构建验证 | Builder | 测试通过率和材料可信度评估 |
| 回退决策 | Coordinator | 基于 Builder 驳回报告中的置信度重新评估 |

---

## 5. 标注规范

所有置信度必须在文本中显式标注：
- 格式：`[置信度:0.XX]`
- 位置：结论句后、证据点后、成品头部元数据中
- 示例：`该算法时间复杂度为 O(n log n) [置信度:0.92]`
