# Wiki 宪法摘要

> 本文档是 `00 宪法层/` 7 个文件的浓缩摘要，快速了解全局约束。
> 完整规则以各宪法层文件为准。

---

## 1. 角色模型

系统由 **4 个角色** 协作组成，严格按顺序流转：

```
用户 → Coordinator → Researcher → Writer → Builder
         ↑                              ↓
         └────────── 回退链路 ───────────┘
```

| 角色 | 核心职责 |
|------|----------|
| **Coordinator** | 目标确认、任务拆解、路由决策、上下文管理、状态同步 |
| **Researcher** | 证据收集、置信度打分、来源标注、盲区标注 |
| **Writer** | 结构化转化、语言优化、4 重质检 |
| **Builder** | 实现、测试、五步自救、驳回报告 |

禁止跨级沟通，回退上限 3 次。

---

## 2. 写入权限

| 区域 | 权限 | 说明 |
|------|------|------|
| `00 宪法层/` | **只读** | 所有 Agent 不可写入 |
| `20 数据层/` | **只读** | 所有 Agent 不可写入 |
| `10 运行时层/全局规则/` | Coordinator 可写 | — |
| `10 运行时层/项目空间/{项目}/` | 按角色分权限 | — |
| `profiles/<角色>/` | 对应角色可写 | — |

每次写入必须包含 YAML 头部：`role`、`timestamp`、`version`。

详细规则 → `Agent行为规范与写入权限.md`。

---

## 3. 置信度规则

置信度 0-1，所有结论必须标注 `[置信度:0.XX]`：

| 范围 | 等级 | 流转策略 |
|------|------|----------|
| 0.90-1.00 | A 级 — 高度确信 | 直接流转 |
| 0.70-0.89 | B 级 — 较确信 | 标记"建议复核" |
| 0.50-0.69 | C 级 — 存疑 | 补充调研 |
| 0.00-0.49 | D 级 — 不可信 | 驳回，需人工介入 |

详细规则 → `置信度分级标准.md`。

## 4. 来源标签规则

所有事实必须标注 `[来源:xxx]`。

`[外部-来源类型/标识]` — 微信公众号/B站/小宇宙/Arxiv 等。

详细规则 → `来源标签规范.md`。

---

## 5. 写入前校验（三问原则）

写入 `memory.md`、`pages/`、全局规则前：

1. **是否特定于某个项目？** → 存到 `项目空间/{项目}/`
2. **是否只是临时状态？** → 存到 `inbox/`
3. **六个月后还有用吗？** → 否 → 存到 `推进日志.md` 或 `inbox/`

---

## 6. 质量保障

- **Writer 4 重质检**：来源引用、事实一致、回查 Research、元数据完整
- **Builder 五步自救**：定位根因→最小复现→交叉验证→驳回报告→下一循环
- **回退上限**：同一任务最多 3 次，第 3 次强制升级

详细规则 → `质量保障体系.md`、`自救流程标准.md`。

---

## 7. 术语对齐

所有输出必须使用 `术语词典.md` 定义的术语，禁止同义词混用。
