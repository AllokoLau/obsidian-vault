# 项目协作规则：PRJ-2026-001 信息挖掘工具

> **层级**: 运行时层 / 项目空间

---

## 1. 本项目角色映射

| 角色 | 负责人 | 特殊说明 |
|------|--------|----------|
| Coordinator | Claude Code | 任务拆解、路由、定时任务管理 |
| Researcher | Claude Code | 技术可行性调研、置信度评估 |
| Writer | Claude Code | Skill 定义文件、模板更新、规范文档 |
| Builder | Claude Code | 抓取脚本、管道搭建 |
| 用户 | 你 | 提供 B站登录凭证、微信公众号链接、确认小宇宙方案 |

## 2. 产出流转规则

- **Researcher** 调研结论 → `inbox/` → Coordinator 审阅
- **Writer** 写作文档 → `inbox/` → Coordinator 确认后入 `outputs/`
- **Builder** 构建产物 → `outputs/`，驳回报告 → `inbox/rebuttal-*.md`
- **Coordinator** 更新 `任务清单.md`、`项目推进日志.md`、`项目决策记录.md`

## 3. 本项目特殊约束

- 所有信息源的技术可行性必须先由 Researcher 出具带置信度的报告
- 置信度评估遵循 `00 宪法层/置信度分级标准.md`
- 来源标签遵循 `00 宪法层/来源标签规范.md`，新来源类型必须先登记
- 音频内容不直接处理（无 STT 能力），只处理文字内容

## 4. 本项目来源偏好

| 来源 | 权威性权重 | 说明 |
|------|-----------|------|
| 微信公众号文章 | 0.85（权威媒体类） | 有编辑审核 |
| B站字幕（AI生成） | 0.70（自动生成） | 可能有识别错误 |
| 小宇宙 RSS shownotes | 0.50（摘要类） | 不完整，置信度上限 0.69 |
| 小宇宙逐字稿 | 0.80（若可获取） | 完整转录 |
