## 7. Dwarkesh Podcast

| 维度 | 详情 |
|------|------|
| **类型** | 英文 Blog / 网站 |
| **平台** | Substack（https://www.dwarkesh.com） |
| **原始域名** | dwarkeshpatel.com |
| **RSS Feed（推测）** | `https://www.dwarkeshpatel.com/feed` |
| **更新频率** | 每周 1-2 篇 |
| **内容方向** | AI 前沿、科技深度访谈、人才与创新 |
| **订阅量** | 8.1 万+ |
| **语言** | 英文（需翻译） |

### 访问方式

**方案 A：Substack API（推荐）**
- Substack 有公开 API 文档：https://substackapi.dev
- 查询最新文章：`GET /api/v1/posts?limit=10`
- 每篇包含：标题、正文、发布日期、作者
- 部分端点可能需要 API key

**方案 B：RSS Feed 抓取**
- Substack 出版物自带 RSS feed
- 格式通常为 `https://{publication}.substack.com/feed`
- 可直接通过 feedparser (Python) 解析

**方案 C：网页抓取**
- `dwarkeshpatel.com` / `dwarkesh.com` 上的文章
- 文章以标准 HTML 格式呈现
- 需处理英文→中文翻译

### 翻译方案

根据 @alloko 确认，使用 **DeepSeek API** 进行翻译：
- 保留原文 + 翻译文本并存
- 标记来源为 [来源-英文/DwarkeshPodcast]

### 访问方式 vs 中文搬运站

已搜索中文搬运站，未发现稳定的 Dwarkesh 中文翻译站点。建议直接抓取原文 + DeepSeek 翻译。

### 置信度评估

## 技术建议

### 推荐抓取工具栈

```python
# 微信公众号 → WeWe RSS（Docker）
# 小宇宙播客 → xyz-dl (Python) 或 RSSHub

```

### 目录结构建议（raw/ 下）

```
raw/
├── 公众号/
│   ├── 海外独角兽/
```

### 注意事项

1. 微信公众号批量抓取需注意频率限制（WeWe RSS 添加公众号频率过高易封控）
2. xyz-dl 使用登录态下载可能触发小宇宙风控，建议免登录模式
3. Dwarkesh 文章通常较长（5000-15000 字），翻译需规划 token 预算
4. 优先启动 RSSHub 路由（微信公众号和小宇宙均有成熟路由）
