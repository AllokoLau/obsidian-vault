# 项目目标书：信息收集管线

## 项目背景
建立自动化信息收集与处理管线，从固定信息源持续采集高质量内容，经置信度评估后入库 Obsidian 知识库。

## 信息源（本期：4个）
1. **晚点聊LateTalk** — 小宇宙播客
2. **厚雪长波** — 小宇宙播客
3. **十字路口Crossing** — 小宇宙播客
4. **张小珺｜商业访谈录** — 小宇宙播客

## 成功标准
- 每日从4个源收集 ≥50 条完整内容
- 内容经 Researcher 置信度标注 → Writer 4重质检 → Builder 自动化入库
- 输出格式：按 Wiki 模板（创始人-姓名 / 知识-主题）

## 约束条件
- 严格遵循 Coordinator → Researcher → Writer → Builder 流转
- 所有事实必须带来源标签和置信度打分
- 置信度 ≥0.7 入库 wiki/ | 0.4-0.69 入 Processing/ | <0.4 删除

## 角色职责
- **@Patricia (Researcher)**: 研究每个信息源的内容结构，制定采集方案和置信度标准
- **@Leclerc (Writer)**: 将研究材料转化为结构化内容，执行4重质检
- **@Kris (Builder)**: 实现自动化采集、处理和入库脚本
