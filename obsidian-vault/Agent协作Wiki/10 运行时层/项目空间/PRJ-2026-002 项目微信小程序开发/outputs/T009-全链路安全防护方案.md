## 二、云函数安全加固方案

### 2.1 createOrder — 事务 + 服务端定价

**安全目标**：
- 防止订单数据和库存不一致（事务）
- 防止客户端篡改价格（服务端定价）
- 防止参数注入（白名单校验）

**核心变更**：

```diff
- 接收: items (含 price), remark
- 直接写入订单 + 扣库存
+ 接收: goodsList (仅 goodsId + quantity), remark
+ 校验: 每个 goodsId 格式 /^[a-f\d]{24}$/
+ 服务端: 从数据库读取 price 重新计算
+ 事务: 订单写入 + 库存扣减在一个事务中
+ 幂等: 订单号使用 时间戳+随机数+openid 后缀
```

**示例接口**：
```javascript
// 请求
{ goodsList: [{ goodsId: "xxx", quantity: 2 }], remark: "少辣" }
// 响应
{ code: 0, data: { orderNo: "ORD20260625001", totalAmount: 59.80 } }
```

### 2.2 updateOrderStatus — 状态机约束

**安全目标**：
- 防止跳级修改订单状态
- 防止修改他人订单

**状态机流转规则**：
```
pending ──→ confirmed ──→ completed
    │                        ↑
    └──→ cancelled ──────────┘
（pending→cancelled 和 confirmed→cancelled 都允许）
```

**核心变更**：
```diff
- 校验 status 在白名单中
+ 读取当前订单状态，检查流转合法性
+ pending 可转为 confirmed / cancelled
+ confirmed 可转为 completed / cancelled
+ completed / cancelled 不可再变更
```

### 2.3 getOrders — 分页 + 字段限制

**安全目标**：
- 防止一次拉取过多数据（性能&配额安全）
- 防止返回不必要字段

**示例接口**：
```javascript
// 请求
{ status: "pending", pageSize: 10, pageNum: 1 }
// 响应
{ code: 0, data: { list: [...], total: 25, hasMore: true } }
```

### 2.4 initGoods — 管理员权限

**安全目标**：
- 只有管理员可重置商品数据

**实现**：
```javascript
// 检查 openid 是否在管理员白名单中
const ADMIN_OPENIDS = ['管理员openid'] // 部署时替换
const wxContext = cloud.getWXContext()
if (!ADMIN_OPENIDS.includes(wxContext.OPENID)) {
  return { code: -403, msg: '无权限' }
}
```

## 五、数据库安全规则

以下规则需在云开发控制台中为各集合手动配置：

### goods（所有用户可读，仅云函数可写）
```json
{ "read": true, "write": false }
```

### cart（仅本人可读写）
```json
{ "read": "doc._openid == auth.openid", "write": "doc._openid == auth.openid" }
```

### orders（仅本人可读，写操作仅通过云函数）
```json
{ "read": "doc._openid == auth.openid", "write": false }
```

> **说明**：安全规则需登录 [微信云开发控制台](https://console.cloud.tencent.com/tcb) → 数据库 → 选择集合 → 权限设置 → 自定义规则 进行操作。

## 六、影响范围与回退

这次的变更会消除 localStorage 下单逻辑，意味着：
- ⚠️ 如果已有测试订单数据在 localStorage，更换后这些数据不可见（从云数据库查询）
- 购物车数据仍在 localStorage（后续可迁移到云数据库）
- 本地 mock 模式将失效，必须配置云环境才能使用

**回退方案**：cloud.js 保留原有 TODO 注释的 mock 代码，如需回退改回 stub 即可。
