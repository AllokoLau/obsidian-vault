## 一、项目架构总览

### 技术栈

| 层级 | 方案 | 说明 |
|------|------|------|
| **前端框架** | 微信原生（WXML + WXSS + JavaScript） | 学习目的，不引入跨端框架 [来源:外部-百度开发者社区/6339972] |
| **组件库** | **TDesign**（腾讯官方组件库） | 提供开箱即用的 UI 组件，与微信风格一致 [来源:外部-百度开发者社区/6339972] |
| **状态管理** | 原生 Page/Component 数据模型 | MVP 阶段不需要全局状态管理库 |
| **后端** | **微信云开发**（云数据库 + 云函数 + 存储） | 免运维，快速验证 MVP [来源:外部-官方文档/微信云开发/计费说明] |
| **编程语言** | JavaScript | 降低入门门槛，后续可迁移到 TypeScript [来源:Writer 基于 Researcher 建议] |
| **IDE** | 微信开发者工具 + VS Code | 双编辑器互补 [来源:Writer 基于 Researcher 建议] |
| **版本控制** | Git | 代码版本管理 |
| **代码规范** | ESLint + Prettier | 保持代码风格统一 [来源:外部-GitHub/Xaviw/weapp-template] |

### 推荐理由

**微信原生开发**是最适合本项目（学习练手 + 单平台）的方案 [置信度:0.88]：

```
跨端框架（Taro / uni-app）的核心价值在"跨端"。
本题只做微信端 → 跨端价值归零，只留下额外成本：
  - 性能损失 10-40%（Taro 首屏慢 ~130ms）
  - 包体积增加 80-200KB
  - 遮蔽微信底层机制（App/Page/Component 生命周期、setData 模型）
  - 调试链路变长
```
[来源:外部-腾讯云开发者社区/2667283] [置信度:0.85]

**微信云开发**提供了零运维的后端方案，对于学习项目是最低摩擦的选型 [置信度:0.95]：
- 免费版 20 万次调用/月、2GB 存储 — 学习项目完全够用
- 云数据库（类 MongoDB）直接在小程序端读写，无需搭建 API 服务器
- 云函数处理需要后端的业务逻辑（如订单生成、库存扣减）
[来源:外部-官方文档/微信云开发/计费说明] [置信度:0.95]

## 二、页面结构与路由

### 页面清单（MVP）

共 **6 个页面 + 1 个底部 Tab 导航**：

| 页面 | 路径 | 底部 Tab | 功能 |
|------|------|----------|------|
| 🏠 **商品列表（首页）** | `pages/index/index` | 1️⃣ | 展示商品卡片列表，点击进入详情 |
| 📄 **商品详情** | `pages/goods/detail` | ❌ | 展示商品图文详情，加入购物车 |
| 🛒 **购物车/点单** | `pages/cart/index` | 2️⃣ | 查看已选商品，调整数量，去结算 |
| 💳 **订单结算** | `pages/order/checkout` | ❌ | 确认订单，模拟结算 |
| 📋 **我的订单** | `pages/order/list` | ❌ | 查看历史订单列表（从"我的"页进入） |
| 👤 **我的** | `pages/user/index` | 4️⃣ | 用户信息、订单入口、设置 |

### 页面关系图

```
底部 Tab（4 个）:
  [商品列表] --- [购物车] --- [订单] --- [我的]
       │              │                   │
       ▼              ▼                   ▼
  [商品详情]     [订单结算]          [我的订单]
```

### 路由策略
- **Tab 页面**: 使用 `app.json` 中 `tabBar` 配置，页面常驻内存
- **非 Tab 页面**: 使用 `wx.navigateTo` 跳转，支持返回
- **结算成功**: 使用 `wx.redirectTo` 重定向（不可返回购物车，避免重复下单）

## 三、云开发数据模型

### 数据库集合设计

MVP 需要 3 个集合：

#### ① `goods` — 商品表

```javascript
{
  "_id": "自动生成",
  "name": "商品名称",           // String
  "price": 29.90,              // Number，单位元
  "image": "cloud://...",      // String，商品图片云存储路径
  "description": "商品描述",   // String，图文描述（RichText）
  "category": "main",          // String，分类（预留：main/side/drink）
  "stock": 100,                // Number，库存
  "status": "on",              // String，上下架状态 on/off
  "createTime": "2026-06-25"   // Date
}
```

#### ② `cart` — 购物车表

```javascript
{
  "_id": "自动生成",
  "_openid": "用户 openid（云开发自动注入）",
  "goodsId": "商品 _id",       // String，关联 goods 表
  "quantity": 2,               // Number，数量
  "selected": true,            // Boolean，是否选中（支持批量结算）
  "updateTime": "2026-06-25"   // Date，更新时间
}
```

> **说明**：云数据库的权限规则设为 `_openid == doc._openid`，每个用户只能读写自己的购物车数据 [来源:外部-官方文档/微信云开发/数据库] [置信度:0.95]

#### ③ `orders` — 订单表

```javascript
{
  "_id": "自动生成",
  "_openid": "用户 openid（云开发自动注入）",
  "orderNo": "ORD20260625001", // String，订单号（云函数生成）
  "items": [                   // Array，订单商品快照
    {
      "goodsId": "商品 _id",
      "name": "商品名称（快照）",
      "price": 29.90,
      "quantity": 2,
      "image": "cloud://..."
    }
  ],
  "totalAmount": 59.80,        // Number，总金额
  "status": "pending",         // String，订单状态
  "remark": "少辣",            // String，备注（预留）
  "createTime": "2026-06-25"   // Date
}
```

**订单状态枚举**: `pending`(待处理) → `confirmed`(已确认) → `completed`(已完成) → `cancelled`(已取消)

[来源:外部-百度开发者社区/7209068] [置信度:0.85]

## 四、云函数 API 设计

### 云函数列表

| 云函数名 | 触发方式 | 功能 |
|----------|----------|------|
| `createOrder` | 小程序端调用 | 创建订单（购物车数据 → 订单 + 清空购物车） |
| `getOrders` | 小程序端调用 | 查询当前用户的历史订单 |
| `updateOrderStatus` | 小程序端调用 | 更新订单状态（如取消订单） |
| `initGoods` | 手动触发（管理端） | 初始化商品数据（首次部署用） |

### 关键云函数示例

**createOrder 核心逻辑**：
```
1. 接收：用户 openid + 购物车选中商品列表
2. 处理：校验库存 → 生成订单号 → 扣减库存
3. 写入：orders 集合插入订单文档
4. 清理：删除购物车中已下单的商品
5. 返回：订单 ID + 订单号
```

> **注意**：MVP 阶段模拟结算，不接入真实微信支付。订单金额仅供展示。
