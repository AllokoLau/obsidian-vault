# 项目协作规则 — 微信小程序开发

## 工作流
按标准四角色流转：Coordinator → Researcher → Writer → Builder

## 角色分配

| 角色 | 主要任务类型 |
|------|-------------|
| Coordinator | 任务拆解、路由、状态同步 |
| Researcher | 技术选型调研、最佳实践研究 |
| Writer | 技术方案文档、架构设计文档、开发指南 |
| Builder | 代码实现、测试、部署 |

## 本项目的特殊规则

1. **原型审核**：第一版 UI 模型需经用户审核通过后再实现
2. **增量开发**：每个功能模块原子化实现，一次只做一个功能点
3. **学习导向**：代码注释要详细，记录关键设计决策和踩坑经验

## 交付物路径
- 研究材料 → `inbox/`
- 设计文档 → `outputs/`
- 代码实现 → `outputs/`
- 驳回报告 → `inbox/`
