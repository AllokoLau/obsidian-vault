# 项目目标书 — 微信小程序开发

## 背景
学习微信小程序开发技术，从零搭建一个可运行的微信小程序，掌握全链路开发流程。

## 成功标准
- [ ] MVP 可运行：实现商品点单 → 下单 → 结算核心闭环
- [ ] 代码托管在 Git，提交规范
- [ ] 产出包含：项目代码 + 开发文档 + 部署说明
- [ ] 可在微信开发者工具模拟器和真机上正常运行

## MVP 功能范围
1. **商品浏览**：展示商品列表、商品详情
2. **购物车/点单**：添加商品到购物车，管理数量
3. **下单结算**：生成订单，模拟结算流程
4. **订单管理**：查看历史订单（可选）

## 约束条件
- 技术栈：微信小程序生态为主，跨端框架可选
- 开发工具：微信开发者工具、VS Code、Git
- 后端方案：微信云开发（免运维）
- 设计工具：Figma（原型阶段需讨论确认）

## 验收标准
1. 代码：通过 Builder 自检（单元测试 + 集成测试 + 边界测试）
2. 文档：包含架构说明、启动指南、API 说明
3. 交付物路径：`outputs/`

## 风险预判
| 风险 | 应对 |
|------|------|
| 微信开发者工具版本兼容性问题 | 使用最新稳定版 |
| 云开发配额限制（免费版） | 了解免费版限制，规划合理使用 |
| 小程序审核不确定性 | 开发版本 + 体验版即可满足学习目的 |

## 非目标（明确不做）
- ❌ 不发布到微信市场（仅开发版/体验版）
- ❌ 不做复杂支付对接（模拟结算）
- ❌ 不做多端发布（先专注微信小程序）
