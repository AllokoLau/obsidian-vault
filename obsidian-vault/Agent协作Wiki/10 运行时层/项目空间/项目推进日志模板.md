# 项目推进日志

> **层级**: 运行时层 / 项目空间  
> **创建者**: Coordinator  
> **更新频率**: 项目内每次状态变化时追加

## 日志

| 时间 | 事件类型 | 任务ID | 角色 | 事件描述 |
|------|----------|--------|------|----------|
| [ISO 8601] | 任务开始 | T001 | Coordinator | 拆解任务并路由到 Researcher |
| [ISO 8601] | 任务完成 | T001 | Researcher | 提交研究结论，置信度 0.85 |
| [ISO 8601] | 决策 | - | Coordinator | 决定采用方案A，因... |
| [ISO 8601] | 回退 | T003 | Builder | BR001 驳回申请，回退到 Researcher |
| [ISO 8601] | 里程碑 | - | Coordinator | 第一阶段完成 |

## 事件类型

| 事件类型 | 说明 |
|----------|------|
| 任务开始 | 新任务启动 |
| 任务完成 | 任务正常完成 |
| 驳回 | Builder 提交驳回报告 |
| 回退 | 因驳回触发回退 |
| 决策 | Coordinator 做出关键决策 |
| 里程碑 | 项目阶段节点 |
| 归档 | 项目关闭 |

## 使用规则

1. 每发生一次状态变化，在日志中追加一行
2. 时间戳精确到分钟
3. 事件描述 1-2 句话，说清楚发生了什么
4. 涉及决策时，在 `项目决策记录.md` 中详细记录
