# 记忆路由总纲

> 规定所有类型信息的唯一存放位置。违反此规则视为信息污染。

---

## 路由总表

| 信息类型 | 存放位置 | 说明 |
|----------|----------|------|
| **角色身份** | `profiles/<角色>/soul.md` | Agent 是谁、做什么、不做什么 |
| **用户偏好** | `profiles/<角色>/USER.md` | 对用户的理解 |
| **角色经验** | `profiles/<角色>/memory.md` | 跨项目可复用的经验（三问校验后写入） |
| **角色技能** | `profiles/<角色>/skills/README.md` | 指向 `SKILL/` 的索引 |
| **宪法层规则** | `00 宪法层/` | 全局不可变规则（**只读**） |
| **数据层资产** | `20 数据层/` | 方法论、模板、规范（**只读**） |
| **项目目标** | `10 运行时层/项目空间/{项目}/项目目标书.md` | 是什么、为什么做、成功标准 |
| **项目上下文** | `10 运行时层/项目空间/{项目}/上下文摘要.md` | 当前状态、活跃任务、已加载规则 |
| **任务状态** | `10 运行时层/项目空间/{项目}/任务清单.md` | Doing / Todo / Done / 回退中 |
| **推进记录** | `10 运行时层/项目空间/{项目}/项目推进日志.md` | 发生了什么、按时间排列 |
| **决策记录** | `10 运行时层/项目空间/{项目}/项目决策记录.md` | 已确定方向和不做什么 |
| **协作规则** | `10 运行时层/项目空间/{项目}/项目协作规则.md` | 本项目内各角色如何协作 |
| **临时材料** | `10 运行时层/项目空间/{项目}/inbox/` | 半成品、草稿、研究材料、未确认结论、驳回报告 |
| **正式产出** | `10 运行时层/项目空间/{项目}/outputs/` | 已确认可交付的内容 |
| **跨项目方法论** | `20 数据层/` | 可复用方法论、模板 |
| **项目状态总览** | `10 运行时层/项目看板.md` | 跨项目 Dataview 聚合 |
| **全局日志** | `10 运行时层/全局规则/Agent协作日志.md` | 所有 Profile 的协作追踪 |
| **技能分配** | `10 运行时层/全局规则/技能分配表.md` | 技能↔Profile 的分配矩阵 |
| **用户偏好（全局）** | `10 运行时层/全局规则/用户偏好记录.md` | 用户整体偏好 |
| **原始资料** | `raw/` | 论文、文章、网页快照（只读不改） |
| **素材** | `assets/` | 图片、截图、架构图 |
| **归档** | `archive/` | 不活跃、过期、废弃内容 |

## 三问校验（写入前必查）

任何 Profile 在写入 `memory.md` 或系统规则文件之前，必须先执行：

1. **这条信息是否特定于某个项目？** → **是**：存到 `10 运行时层/项目空间/{项目}/`，不存 profiles/
2. **这条信息是否只是临时状态？** → **是**：存到 `inbox/`，不存正式位置
3. **这条信息六个月后还有用吗？** → **否**：存到 `项目推进日志.md` 或 `inbox/`，不存 memory/ 或全局规则

**违反任意一条即为信息污染，应立即纠正。**

---

## 信息污染预警

| 污染现象 | 说明 |
|----------|------|
| `soul.md` 里出现项目名 | 角色身份被项目状态污染 |
| `memory.md` 里出现"今天做了什么" | 角色经验被项目进度污染 |
| 全局规则中出现单次项目笔记 | 临时想法污染长期规则 |
| `outputs/` 里出现草稿 | 未验证材料混入正式产出 |

## 跨角色流转规则

| 角色 | 读取范围 | 写入范围 |
|------|----------|----------|
| **Coordinator** | 全部 | `项目目标书`、`任务清单`、`上下文摘要`、`项目推进日志`、`项目决策记录`、`全局规则/`、项目看板 |
| **Researcher** | 全部 | `inbox/`（研究材料） |
| **Writer** | 全部 | `inbox/`（写作成品初稿）、`outputs/`（确认后） |
| **Builder** | 全部 | `outputs/`（构建产物）、`inbox/`（驳回报告） |

> 详细写入权限见 `Agent行为规范与写入权限.md`
