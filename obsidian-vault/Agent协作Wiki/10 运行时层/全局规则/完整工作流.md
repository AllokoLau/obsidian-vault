# 完整工作流

> 用户发布任务后的端到端流转链路。Coordinator 全程驱动。

---

## 流程图

```
用户发布任务
    │
    ▼
┌─────────────────────────────────────────┐
│  1. Session Gate                        │
│  确认项目归属，加载项目上下文            │
└────────────────┬────────────────────────┘
                 │
                 ▼
┌─────────────────────────────────────────┐
│  2. 任务拆解                             │
│      ↓                                  │
│  写入 任务清单.md (Todo)                 │
└────────────────┬────────────────────────┘
                 │
                 ▼
┌─────────────────────────────────────────┐
│  3. 路由到 Researcher                    │
│      ↓                                  │
│  Researcher 调研 → 产出附带置信度评分    │
│  产出 → inbox/                          │
│      ↓                                  │
│  Coordinator 检查置信度                  │
│  ├─ > 80% → 下一步                      │
│  ├─ 60-80% → 标注不确定点，继续          │
│  ├─ 40-60% → 双盲交叉验证               │
│  └─ < 40% → 需人工介入                  │
└────────────────┬────────────────────────┘
                 │
                 ▼
┌─────────────────────────────────────────┐
│  4. 路由到 Writer                        │
│      ↓                                  │
│  Writer 转化 → 4 重质检                   │
│  ├─ 质检通过 → 产出 → outputs/          │
│  └─ 质检不通过 → 回退到 Researcher       │
└────────────────┬────────────────────────┘
                 │
                 ▼
┌─────────────────────────────────────────┐
│  5. 路由到 Builder (仅代码/配置类任务)    │
│      ↓                                  │
│  Builder 实现 → 测试 → 五步自救           │
│  ├─ 通过 → 交付 → outputs/              │
│  └─ 材料错误 → 驳回报告 → inbox/        │
│       ↓  Coordinator 重新路由            │
│       ├─ 事实错误 → 回退 Researcher       │
│       ├─ 转化错误 → 回退 Writer          │
│       └─ 拆解不当 → Coordinator 重拆      │
│  回退上限 3 次，第 3 次强制升级            │
└────────────────┬────────────────────────┘
                 │
                 ▼
┌─────────────────────────────────────────┐
│  6. 汇总与归档                           │
│      ↓                                  │
│  Coordinator 汇总结果 → 更新:            │
│  ├─ 任务清单.md (Todo→Done)             │
│  ├─ 项目仪表盘.md                        │
│  ├─ Agent协作日志.md                     │
│  ├─ 项目推进日志.md                      │
│  └─ 决策记录.md (如有决策变更)           │
└─────────────────────────────────────────┘
```

---

## 关键规则

1. **未验证材料不进正式产出**：Researcher 和 Writer 的产出放 `inbox/`，只有 Builder 的交付放 `outputs/`
2. **禁止跨级**：Researcher ↔ Builder 不能直接对话，必须经 Coordinator
3. **置信度门槛**：Researcher 置信度 < 0.4 的结论不进入下一环节
4. **回退上限 3 次**：同任务回退 ≥ 3 次 → 强制升级给用户
5. **每个角色完成后提交完成报告**：由 Coordinator 决定下一步
