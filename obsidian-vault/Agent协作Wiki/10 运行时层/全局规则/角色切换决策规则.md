# 角色切换决策规则

> 换项目不一定换 Profile，换角色才需要换 Profile。

---

## 决策树

```
用户提出新任务
    │
    ▼
这次变化是角色变化吗？
    │
    ├── YES ──→ 职责是否变化？
    │               │
    │               ├── YES ──→ 切换 Profile
    │               │             （如从 Researcher 切换到 Writer）
    │               │
    │               └── NO  ──→ 保留当前 Profile
    │                             （如同是 Researcher，只是换证据类型）
    │
    └── NO  ──→ 只是项目变化？
                    │
                    ├── YES ──→ 切换 Project context
                    │             （加载新项目空间中的文件）
                    │
                    └── NO  ──→ 继续当前会话
                                  （同一项目同一角色，继续推进）
```

---

## 示例

| 场景 | 判定 | 动作 |
|------|------|------|
| 信息挖掘项目完成，新启动"建个人网站"项目 | 角色不变 (Coordinator)，项目变化 | 切换 Project context，加载新项目文件 |
| Coordinator 做完任务拆解，需要 Researcher 做调研 | 角色变化 (Coordinator→Researcher)，职责变化 | 切换 Profile |
| 用户说"继续上个任务" | 角色不变，项目不变 | 继续当前会话 |
| Researcher 做完一期调研，用户说"再深挖这个方向" | 角色不变 (仍是 Researcher)，职责不变 | 保留当前 Profile |

---

## 上下文加载规则

切换时，Coordinator 必须确保以下上下文被正确加载：

| 切换类型 | 应加载 | 不应加载 |
|----------|--------|----------|
| 切换 Profile | 新角色的 soul.md + USER.md + memory.md + 对应技能 | 旧角色的角色记忆和技能 |
| 切换 Project | 新项目的 项目目标书.md + 项目协作规则.md + 当前状态 | 旧项目的推进日志和决策记录 |
| 继续当前会话 | 当前项目的 上下文摘要.md + 活跃任务状态 | 无关历史 |
