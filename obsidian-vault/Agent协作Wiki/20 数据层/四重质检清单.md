# 四重质检清单

> **层级**: 数据层 / 质检清单模板  
> **使用角色**: Writer  
> **作用**: Writer 交付前的标准化质检工具

---

## 使用说明

Writer 完成内容转化后，必须逐项执行以下 4 重质检。**全部通过后方可交付。**

---

## 第一重: 来源引用检查

### 检查目标
确保所有事实性陈述都有可追溯的来源标注。

### 检查方法
1. 全文扫描，识别所有事实性陈述（数据、定义、结论、引用）
2. 确认每个事实后是否有 `[来源:XXX]` 标注
3. 检查来源标签格式是否符合 `来源标签规范.md`

### 检查清单
- [ ] 每个数据点都有来源
- [ ] 每个定义都有来源
- [ ] 每个结论都有来源
- [ ] 每个引用都有来源
- [ ] 来源标签格式正确
- [ ] 无"我记得"、"众所周知"等无来源表述

### 通过标准
来源引用覆盖率 = 100%

### 不通过处理
- 补充缺失的来源
- 无法补充来源的陈述 -> 删除或改为"推测"（并降低置信度）

---

## 第二重: 事实一致检查

### 检查目标
确保转化后的内容与 Researcher 原始材料完全一致，无扭曲、无遗漏、无添加。

### 检查方法
1. 逐条对照 Researcher 的 `证据链` 表格
2. 确认每个证据在成品中都有对应
3. 确认证据的置信度未被改变
4. 确认冲突点的处理方式与 Researcher 一致
5. 确认盲区被保留（未被隐藏或忽略）

### 检查清单
- [ ] 所有证据链条目在成品中有对应
- [ ] 证据内容未被扭曲（逐字比对关键数据）
- [ ] 置信度未被无依据提升
- [ ] 冲突点的处理方式与 Researcher 一致
- [ ] 盲区在成品中明确说明
- [ ] 无 Researcher 材料中不存在的新事实

### 通过标准
完全映射，无扭曲、无遗漏、无添加

### 不通过处理
- 逐条修正映射错误
- 删除自行添加的内容
- 补充遗漏的内容

---

## 第三重: 自动回查 Research

### 检查目标
反向检索 Wiki，确认是否有 Researcher 未覆盖的新信息。

### 检查方法
1. 提取成品中的核心结论和关键命题
2. 在 Wiki 数据层/原始素材/ 中检索相关主题
3. 在 Wiki 数据层/方法论/ 中检索是否有更新的方法论
4. 在运行时层/项目空间/ 中检索是否有其他相关任务的材料
5. 在运行时层/全局规则/驳回报告/ 中检索是否有已知错误

### 检查清单
- [ ] 核心结论在 Wiki 中无矛盾的新信息
- [ ] 数据层方法论为最新版本
- [ ] 无其他相关任务的补充材料被遗漏
- [ ] 无历史驳回报告指向当前材料的已知错误
- [ ] 外部来源在成稿后无重大更新（如有时效性要求）

### 通过标准
无重大信息差

### 不通过处理
- 发现新信息 -> 补充到成品中，或标注为"新增信息"并更新置信度
- 发现矛盾信息 -> 回退到 Researcher 重新评估

---

## 第四重: 标注输出成品

### 检查目标
确保成品头部元数据完整、规范。

### 检查方法
对照 `输出成品头部模板.md` 逐项检查

### 检查清单
- [ ] title 已填写
- [ ] version 已填写（从 1.0 开始）
- [ ] project 已填写
- [ ] project_id 已填写
- [ ] research_source 已填写（对应 Researcher 任务ID）
- [ ] quality_check 4 项全部 ✓
- [ ] overall_confidence 已填写（不高于 Researcher 的 overall_confidence）
- [ ] confidence_basis 已填写
- [ ] produced_by 已填写
- [ ] produced_at 已填写（ISO 8601 格式）
- [ ] target_audience 已填写
- [ ] language 已填写

### 通过标准
元数据完整，格式正确

### 不通过处理
- 补充缺失的元数据
- 修正格式错误

---

## 质检记录

完成 4 重质检后，填写以下记录:

```yaml
---
check_id: Q001
timestamp: [ISO 8601]
artifact_id: [成品ID]
checker: Writer
---

## 质检结果
- ① 来源引用: [通过/不通过] [覆盖率:XX%]
- ② 事实一致: [通过/不通过]
- ③ 自动回查: [通过/不通过] [新发现:有/无]
- ④ 标注成品: [通过/不通过]

## 整体结果
- [ ] 全部通过，可交付
- [ ] 存在不通过项，返回修改

## 备注
[如有不通过项，记录修改计划]
```
