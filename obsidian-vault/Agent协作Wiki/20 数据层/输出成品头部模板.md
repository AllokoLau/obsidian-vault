# 输出成品头部模板

> **层级**: 数据层 / 写作模板  
> **使用角色**: Writer  
> **作用**: 标准化所有输出成品的元数据头部

---

## 标准头部

```markdown
---
title: [标题]
version: 1.0
project: [项目名]
project_id: [PRJ-YYYY-NNN]
research_source: [Researcher 任务ID，如 T001]
quality_check:
  - source_citation: ✓
  - factual_consistency: ✓
  - research_backcheck: ✓
  - metadata_complete: ✓
overall_confidence: [0.XX]  # 取 Researcher 整体置信度与 Writer 质检的综合
confidence_basis: [综合方式说明，如 "取 Researcher 0.85 与回查无差异，维持 0.85"]
produced_by: Writer
produced_at: [ISO 8601]
last_updated: [ISO 8601]
word_count: [字数]
target_audience: [受众，如 "技术团队" / "管理层" / "外部客户"]
language: [zh-CN / en-US]
---
```

## 可选扩展字段

根据输出类型，可添加以下字段:

### 文档类型
```yaml
doc_type: 技术文档 / 产品文档 / 分析报告 / 方案设计 / 用户手册
```

### 代码相关
```yaml
code_language: [Python / JavaScript / Go / ...]
framework: [React / Vue / Django / ...]
test_coverage: [X%]
```

### 审批相关
```yaml
review_status: 待评审 / 评审中 / 已通过
reviewer: [评审人]
approved_at: [ISO 8601]
```

### 版本历史
```yaml
version_history:
  - version: 1.0
    date: [ISO 8601]
    change: 初始创建
    author: [Writer]
```

## 使用规则

1. **必须字段**: title, version, project, research_source, quality_check, overall_confidence, produced_by, produced_at
2. **置信度规则**: 不得高于 Researcher 的 overall_confidence，除非回查发现了支撑性新信息（此时需标注）
3. **质检规则**: 4 项质检必须全部 ✓ 才能交付
4. **更新规则**: 每次修改后更新 version 和 last_updated
