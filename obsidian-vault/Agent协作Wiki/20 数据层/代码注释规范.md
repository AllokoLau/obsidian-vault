# 代码注释规范

> **层级**: 数据层 / 代码模板  
> **使用角色**: Builder  
> **作用**: 标准化代码注释和文档，确保可维护性和可追溯性

---

## 1. 文件头部注释

每个代码文件顶部必须包含:

```python
"""
File: [文件名]
Project: [项目名] ([PRJ-YYYY-NNN])
Author: Builder
Created: [ISO 8601]
Last Updated: [ISO 8601]

Description:
    [文件功能描述，2-3句话]

Dependencies:
    - [依赖1]
    - [依赖2]

Research Source:
    - [Writer 成品ID / Researcher 任务ID]
    - [相关来源标签]

Quality Check:
    - Unit Test: ✓/✗
    - Integration Test: ✓/✗
    - Boundary Test: ✓/✗

Confidence: [0.XX]
"""
```

## 2. 函数/方法注释

```python
def function_name(param1: type, param2: type) -> return_type:
    """
    [一句话功能描述]

    Args:
        param1: [参数说明] [来源:XXX]
        param2: [参数说明]

    Returns:
        [返回值说明]

    Raises:
        [异常类型]: [触发条件]

    Example:
        >>> function_name(1, 2)
        3

    Research Source:
        - [算法/逻辑来源] [来源:XXX]

    Confidence: [0.XX]
    """
```

## 3. 关键逻辑注释

对于复杂逻辑或关键决策点:

```python
# 关键决策点: [决策描述]
# 依据: [Writer 成品中的相关段落] [来源:XXX]
# 置信度: [0.XX]
# 替代方案: [如果有，简述为什么不选]
```

## 4. 配置注释

```yaml
# Project: [项目名]
# Source: [Writer 成品ID]
# Confidence: [0.XX]
# Last Updated: [ISO 8601]

key: value  # [配置项说明] [来源:XXX]
```

## 5. 测试注释

```python
def test_feature():
    """
    Test: [测试功能]
    Source: [Writer 成品中的验收标准] [来源:XXX]
    Expected: [预期行为]
    Boundary: [边界条件]
    """
```

## 6. 驳回标记

当代码中包含已知待修复的问题时:

```python
# TODO-BUILDER-BACKLOG: [驳回报告ID]
# Issue: [问题描述]
# Status: 等待新材料 / 已回退到 Researcher
# Expected Fix: [预期修复方式]
```
