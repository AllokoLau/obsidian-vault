# Skill: 信息挖掘

> 从指定信息源采集内容，经过置信度评估后存入 Obsidian 知识库（raw → process → wiki）

---

## 1. 触发条件

### 自动触发
- **B站**：Cron 定时检查指定 UP 主的新视频
- **小宇宙**：Cron 定时拉取 RSS 更新
- **微信公众号目录**：Cron 定时扫描新链接文件

### 手动触发
- 用户直接投递内容到 `raw/`
- 用户提供公众号链接

---

## 2. 工作流

```
输入 → [Source Adapter] → 统一格式
     → [Researcher] → 置信度评估
     ├─ < 0.4 → 丢弃
     ├─ 0.4-0.7 → process/（等人为判断）
     └─ > 0.7 → wiki/（直接入库）
     → [Coordinator] → 更新任务记录
```

### 2.1 Source Adapter（各来源转统一格式）

| 来源 | 适配方式 | 输出格式 |
|------|---------|---------|
| B站视频 | bilibili-api-python 获取字幕 | Markdown 全文 |
| 小宇宙 RSS | RSSHub 拉取 | Markdown（shownotes） |
| 微信公众号 | wechat-exporter 导出 | Markdown 全文 |
| 用户手动提交 | 直接读取 | 原格式 |

### 2.2 统一格式

所有入库内容经过 Source Adapter 后统一为：

```yaml
---
type: knowledge          # knowledge / founder
入库日期: YYYY-MM-DD
置信度: 0.00            # Researcher 填写
来源: [来源标签]        # 遵从 来源标签规范.md
主题: 主题名称
分类: 分类名
tags: []
---

## 核心结论
...

## 详细内容
...

## 来源
- [来源标签] 原文链接
```

### 2.3 置信度评估（Researcher）

遵循 `00 宪法层/置信度分级标准.md`：

| 置信度区间 | 路由 | 动作 |
|-----------|------|------|
| 0.90 - 1.00 | wiki/ | 直接入库 |
| 0.70 - 0.89 | wiki/ | 标记"建议复核" |
| 0.50 - 0.69 | process/ | 等待人为判断 |
| 0.40 - 0.49 | process/ | 等待人为判断 |
| 0.00 - 0.39 | 丢弃 | 记录日志 |

### 2.4 输出格式

#### wiki/ 入库条目
符合 `wiki/入库模板/` 的格式，YAML 头含定量置信度。

#### process/ 待判断条目
同 wiki 格式，但附加标记：
```markdown
> **状态**: 待人为判断
> **置信度**: 0.55
> **原因**: 仅基于 RSS shownotes，缺乏完整内容
```

## 3. 来源标签规范

所有入库内容的来源必须使用以下标签格式：

| 来源 | 标签格式 | 示例 |
|------|---------|------|
| 微信公众号 | `[外部-微信公众号/文章标题]` | `[外部-微信公众号/AI投资趋势]` |
| B站视频 | `[外部-Bilibili/BV号]` | `[外部-Bilibili/BV1ykA3zPEoD]` |
| 小宇宙播客 | `[外部-小宇宙/播客名/期号]` | `[外部-小宇宙/张小珺/E123]` |
| RSSHub | `[外部-RSSHub/来源名]` | `[外部-RSSHub/晚点聊]` |

## 4. 质量保障

### 4.1 置信度降级规则
- 仅基于 shownotes（无逐字稿）→ 最高 0.69
- 来源未标注 → 最高 0.49
- 冲突点未处理 → 最高 0.69

### 4.2 入库前检查（Writer 4-quality-check）
1. **来源完整性**：所有事实是否有来源标签
2. **置信度标注**：是否有显式 `[置信度:0.XX]` 标注
3. **模板合规**：是否按对应模板填写
4. **YAML 头部**：是否包含 role/created_at/version/project_id
