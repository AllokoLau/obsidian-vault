\# RESTful API 设计规范

\- \*\*基础路径\*\*：统一使用 `/api/v1/` 前缀。

\- \*\*资源命名\*\*：使用复数名词，如 `/api/v1/users`、`/api/v1/orders`。

\- \*\*HTTP 状态码\*\*：遵循标准语义（200成功, 201创建, 400客户端错误, 401未授权, 404不存在, 500服务端错误）。

\- \*\*请求与响应\*\*：请求参数推荐使用 `snake\_case`；响应 JSON 统一封装字段：

&#x20; ```json

&#x20; {

&#x20;   "status": "success" | "error",

&#x20;   "code": 200,

&#x20;   "message": "操作提示",

&#x20;   "data": {} 

&#x20; }
